## 谱面渲染核心 — note 系统推进

### 渲染管理器优化
- 帧序列改为存 `output_dir/frames/` 子文件夹，每次渲染开始清空重建
- 加 `_delete_dir_recursive()` 递归删除辅助函数（修正 `get_dirs()`→`get_directories()`）

### Note 系统
- **多押判断**：`world._ready()` 中排序 `Globals.all_notes`，用相邻比较标记 `multihit`
  - 修正了 `range(1, size()-1)` 跳过两头、else 分支覆盖多押等 bug
- **spawn_notes()**：调用 `set_type(type, multihit)` 设置贴图/高亮，hold 独立实例化
- **`update_notes_position()`**：每帧更新 note 的 Y 坐标
  - Y 单位：1Y = 0.6H，公式 `(floorPosition - floor_position) * window_size.y * 0.6 * above_dir * speed`
  - `above` 用 ±1 区分方向：notes_above 从正Y→0，notes_below 从负Y→0
- **note.speed 两种含义**：
  - 普通 note：视觉速度倍率，直接乘
  - Hold：锁定判定时的线速度，`set_length` 里用了但**视觉位移不能重复乘**
- **透明度事件**：`self.modulate.a` → `$shape.modulate.a`，防止 note 跟线一起变透明
